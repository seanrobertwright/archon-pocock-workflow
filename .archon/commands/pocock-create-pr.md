# Create PR

**Workflow ID**: $WORKFLOW_ID

## Phase 1: LOAD

Read `$ARTIFACTS_DIR/tickets.md`, `$ARTIFACTS_DIR/review/resolution.md`, `$ARTIFACTS_DIR/validation.md`, and `$ARTIFACTS_DIR/postmortem.md` (whichever exist — a bug-fix run has a postmortem instead of tickets; include its confirmed hypothesis and prevention note in the PR body). Determine the parent spec issue: from $ARGUMENTS, or the tickets artifact. Check for a PR template at `.github/PULL_REQUEST_TEMPLATE.md` and follow it if present.

## Phase 2: PUSH + CREATE

Push the current branch. Create the PR against `$BASE_BRANCH` with `gh pr create`:

- Title: from the spec/feature, in the project's domain vocabulary (`CONTEXT.md`)
- Body: what shipped (per ticket), `Closes #<n>` for each completed ticket and a reference to the parent spec issue, the two-axis review summary (Standards / Spec, kept separate), validation results, and a **Deferred findings** section from the resolution artifact
- Every AI-generated issue comment this workflow posts starts with `> *This was generated by AI during an Archon workflow run.*`

## Phase 3: REPORT

Write `$ARTIFACTS_DIR/.pr-url` containing only the PR URL and `$ARTIFACTS_DIR/pr-body.md` with the body you used. End your output with the PR URL.
