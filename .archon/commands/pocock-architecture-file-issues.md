# File Architecture Candidates

**Workflow ID**: $WORKFLOW_ID

## Phase 1: LOAD

Read `$ARTIFACTS_DIR/candidates.md` and `docs/agents/triage-labels.md`. Check existing open issues labeled `architecture-candidate` — do not file duplicates of candidates already on the tracker (match by module/seam, not exact wording). Also skip any candidate that a `.out-of-scope/` entry or an ADR already rejects.

## Phase 2: FILE

For each `Strong` candidate (and `Worth exploring` ones if there are fewer than 2 strong): create a GitHub issue with `gh issue create`:

- Label: `architecture-candidate` (create with `gh label create architecture-candidate --force` first). Do NOT apply `ready-for-agent` — per the `ask-matt` flow, an architecture candidate is an **idea**, and ideas enter the main flow through an interactive `/grill-with-docs` session, not straight to implementation.
- Body starts with `> *This was generated by AI during an Archon architecture-health run.*`, then the candidate's Problem / Solution / Benefits in `CONTEXT.md` + `codebase-design` vocabulary, a pointer to the run's `architecture-review.html` artifact, and a closing line: "Next step: take this into `/grill-with-docs`."

`Speculative` candidates are not filed — list them in the report only.

## Phase 3: REPORT

End your output with: issues created (numbers + titles), duplicates skipped, and speculative candidates left unfiled.
